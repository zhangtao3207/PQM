# 模块连接关系图绘制与排版要求

## 1. 总体目标

模块连接关系图用于清晰展示各模块之间的层级关系、输入输出接口以及信号流向。  
绘制与排版时应优先保证结构清楚、逻辑准确、层级明确、阅读顺序统一，不应引入无实际意义的图形元素或杂乱格式。

## 2. 整体格式要求

### 2.1 文档整体格式要求

- 文档标题、章节标题、子章节标题应层级清晰。
- 标题格式应统一，推荐使用：
- `#` 一级标题
- `##` 二级标题
- `###` 三级标题
- 正文说明应简洁明确，避免口语化表达。
- 同一类内容应采用统一的排版方式，不应混用多种风格。

### 2.2 图文组织要求

- 每张图前应有对应的小节标题或说明文字。
- 图前应说明该图的作用，例如“模块连接关系”“接收流程”“发送流程”。
- 图后可补充简短说明，对图中关键关系进行解释。
- 图与图之间、图与正文之间应留有清晰分隔，避免连续堆叠。

### 2.3 图形布局要求

- 图的整体布局应尽量规则、对齐、紧凑。
- 推荐整体采用从左到右的阅读方向。
- 模块排列应体现主从关系、包含关系和信号流向。
- 同一层级的模块应保持风格一致、位置协调。
- 不应为了视觉填充加入无意义节点或图形。

### 2.4 命名格式要求

- 模块名应使用代码中的实际模块名。
- 信号名应使用代码中的实际端口名。
- 模块名、信号名推荐使用反引号包裹，或在图中直接保持代码命名形式。
- 中英文之间的风格应统一，不应出现同类对象有的翻译、有的保留原名的混乱情况。

## 3. 基本绘制要求

### 3.1 模块表示要求

- 每个模块必须使用一个矩形表示。
- 如果共有 `X` 个模块，则图中只能出现 `X` 个用于表示模块的矩形。
- 不允许为了排版、装饰或占位额外增加无意义矩形。

### 3.2 顶层模块表示要求

- 如果图中存在顶层模块，则顶层模块必须使用一个大矩形表示。
- 顶层模块内部应包含其直接下级子模块。
- 子模块必须绘制在顶层模块矩形内部，以体现包含关系。
- 不允许将“顶层模块”和“其内部子模块”画成相互独立的并列结构。

### 3.3 输入输出布局要求

- 模块输入信号统一放在模块左侧。
- 模块输出信号统一放在模块右侧。
- 输入和输出不得混合摆放。
- 整体信号流向应尽量保持从左到右。
- 同类型端口位置应尽量保持对齐。

## 4. 端口标注要求

### 4.1 单个信号标注

- 单个输入或输出信号可直接标注信号名。
- 信号名应与代码中的端口名保持一致。

### 4.2 多个信号标注

- 当一个位置需要表示多个输入信号或多个输出信号时，信号之间不得使用 `/` 连接。
- 多个信号必须使用换行方式分开展示。
- 信号排列应清晰、对齐，便于阅读。

示例：

```text
tx_en
tx_data
```

而不是：

```text
tx_en / tx_data
```

## 5. 连接关系要求

- 模块之间的连接线应体现真实的信号流向。
- 连接关系应与实际代码中的模块实例化关系一致。
- 上层模块与子模块之间的连接应优先体现“输入到哪里、输出来自哪里”。
- 不应增加与代码无关的虚构连接。
- 连接线应尽量简洁，避免无意义交叉。

## 6. 图形简化要求

- 图中只保留必要元素：模块、输入、输出、连接线。
- 不使用无实际意义的中间节点。
- 不为了视觉对称增加额外占位图形。
- 在保证准确的前提下，图形应尽量简洁。

## 7. 阅读性要求

- 图的阅读方向应统一，推荐从左到右。
- 同一类信息应采用相同表现形式。
- 模块名称、端口名称、连接方向应保持一致风格。
- 图应使读者能够快速看出：
- 有哪些模块
- 哪个是顶层模块
- 各模块的输入输出分别是什么
- 模块之间如何连接
- 示例：FPGA\rtl\uart\uart.md
